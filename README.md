# End-to-End Hospitality Data Analysis using Python

## 📌 Project Overview

This project analyzes hotel booking data to uncover valuable insights into revenue, occupancy, customer ratings, and booking trends. Through data cleaning, feature engineering, and exploratory data analysis, the project provides a better understanding of hotel performance and customer behavior.

---

## 🎯 Project Objective

The objective of this project is to analyze hotel booking data and identify:

- Revenue performance across cities and hotel categories
- Occupancy trends
- Customer ratings
- Booking platform contributions
- Hotel booking behavior

The insights derived from this analysis can help improve pricing strategies, room inventory management, marketing efforts, and operational planning.

---

## 📂 Dataset

The analysis uses the following datasets:

- `fact_bookings.csv`
- `fact_aggregated_bookings.csv`
- `dim_hotels.csv`
- `dim_rooms.csv`
- `dim_date.csv`
- `new_data_august.csv`

## 💡 Key Business Insights

- Mumbai generated the highest realized revenue among all cities.
- RT4 room category contributed the highest revenue.
- Weekend occupancy was higher than weekday occupancy.
- Luxury hotels recorded higher occupancy rates than business hotels.
- Online Travel Agencies (OTAs) and direct online bookings contributed significantly to total revenue.
- The analysis highlights opportunities to improve occupancy and optimize pricing strategies across different cities and hotel categories.
